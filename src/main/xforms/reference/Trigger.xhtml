The trigger Control
Description

This form control is similar to the HTML element button and allows for user-triggered actions. This form control may also be used to construct other custom form controls.

XForms 1.1 Links
Recommendation
Quick Reference
Minimal trigger Control

A trigger with minimal appearance is rendered as a Link.

XForms Markup

<!--XForms UI-->
<xf:trigger id="trigger1" ref="item" appearance="minimal">
    <xf:label>A minimal trigger</xf:label>
    <xf:message level="modal">Clicked minimal trigger</xf:message>
</xf:trigger>

Stanaard trigger

A trigger without any appearance is rendered as a standard button.

XForms Markup

<!--XForms UI-->
<xf:trigger id="trigger2" ref="item">
    <xf:label>A standard trigger</xf:label>
    <xf:message level="ephemeral">Clicked standard trigger</xf:message>
</xf:trigger>

Image trigger

A trigger can reference a picture to display instead of the label.

XForms Markup

<!--XForms UI-->
<xf:trigger id="trigger3" src="../../images/bf_logo_67x27.png" ref="item">
    <xf:label>A image trigger</xf:label>
    <xf:message level="modal">Message from image trigger</xf:message>
</xf:trigger>

Image trigger with CSS

A trigger can be styled by CSS. Here the trigger is given an id and a CSS matcher is added for it. The corresponding button rendered in HTML will have the original id with '-value' attached. The matcher for the button is therefore #imageTrigger-value. This is only one example how to style controls with CSS. Besides identity matching alternative style rules are possible to achieve the same result. E.g. instead of #imageTrigger .xfValue would match the same element in the tree.

XForms Markup
<style type="text/css">
    #trigger4-value{
        background:url(../../images/bf_logo_67x27.png) no-repeat center;
        width:100px;
        height:40px;
    }
</style>
<!--XForms UI-->
<xf:trigger id="trigger4" ref="item">
    <xf:label></xf:label>
    <xf:message level="modal">Message from CSS image trigger</xf:message>
</xf:trigger>
